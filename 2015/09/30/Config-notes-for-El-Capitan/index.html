Config notes for El Capitan

Sep 30, 2015

The new release of OS X, El Capitan, brought several new features, one of them is called rootless, which sets more limits for users. For example, some system directories are no longer allowed to modified even for root user.

However, according to this post, Disabling System Integrity Protection, you can temporarily disable rootless with a restart. For detail steps please check the post above.

In the same time, gem install no longer works with rootless policy, since gem needs the I/O right for /Library. In this post, Can’t install gems on MacOS X “El Capitan”, two solutions are raised:

  • Keep using sudo and relocate binary file to /usr/local/bin
  • Install in your home directory without sudo

Personally I’d suggest the second choice, sudo always troubles me when I’m not certain what it will do and what it has done to me.
